# Training security risks

Tracked risks: prompt injection, malicious samples, private-data leakage, cross-user contamination, test-fixture leakage, benchmark contamination, model-generated-data collapse, duplicate synthetic data, feedback amplification, reward hacking, evaluator bias, and version mismatch.

Contamination flags block dataset approval. Future persisted lineage must connect source record, dataset version, training run, affected model, and deployment version. Phase 0 creates no user-derived shared data.
